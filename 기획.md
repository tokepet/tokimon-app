## 1. 시작하며

요즘 개발자는 하루에 수만에서 수십만 개의 토큰을 AI에게 쓴다. 그런데 그 숫자가 무슨 의미인지 잘 와닿지 않는다. "오늘 10만 토큰 썼다"는 말은 마치 "오늘 6,500보 걸었다"처럼 추상적이다.

TokiMon은 그 숫자를 **살아있는 펫의 성장**으로 바꾼다. 데스크탑 한쪽에 작은 펫이 떠 있고, 사용자가 AI를 쓸 때마다 펫이 자란다. Bongo Cat이 키보드 입력에 반응한다면, TokiMon은 토큰 흐름에 반응한다.

---

## 2. 한 줄로 요약하면

**Claude · OpenAI(그리고 OpenRouter · Together · Ollama 같은 호환 서비스)의 토큰 사용량을 한 곳에 모아, 그에 반응해 성장하고, 누적 토큰을 재화 삼아 새 펫을 뽑을 수 있는 크로스플랫폼 데스크탑 펫.**

Google Gemini는 다음 단계에서 합류한다.

## 3. 어떤 문제를 푸는가

| 사용자가 느끼는 불편 | TokiMon이 주는 것 |
|---|---|
| AI를 얼마나 썼는지 숫자로 봐도 와닿지 않음 | 펫의 표정·진화로 사용량을 **체감** |
| 토큰 비용이 "사라지는 돈"으로만 느껴짐 | 토큰이 **펫을 키우는 재화**가 되어 보상이 된다 |
| 데스크탑이 작업 도구로만 가득함 | 항상 떠 있는 **정서적 동반자** |
| AI 작업의 누적 성과가 보이지 않음 | 펫의 성장이 **시각적 포트폴리오** |

이 네 가지 중 어느 하나에 공감한다면 TokiMon의 타겟이다.

## 4. 누구를 위해 만드는가

**1차 타겟**은 AI API를 일상적으로 호출하는 개발자다. 구체적으로는:

- Claude API · OpenAI API를 직접 호출하는 사람
- Claude Code CLI를 매일 쓰는 사람
- OpenRouter · Together · Groq 같은 호환 서비스를 쓰는 사람
- Ollama · LM Studio로 로컬 LLM을 돌리는 사람

**2차 타겟**은 AI 사용 패턴을 시각화하고 싶은 인디 해커와 학생.

**3차 (확장)** 타겟은 Gemini 사용자(Phase 1.5)와 팀 단위로 운영하는 스타트업·스튜디오.

## 5. 꼭 지켜야 할 제약

타협하지 않을 6가지를 먼저 못 박는다. 새 기능 제안이 들어오면 이 목록과 부딪치는지부터 확인한다.

1. **상시 노출** — 데스크탑 위에 떠다니는 윈도우. Bongo Cat 스타일.
2. **크로스플랫폼** — macOS와 Windows 동시 지원. Linux는 가능한 범위에서.
3. **저메모리** — 유휴 시 80MB 이하 (v1, Python 사이드카 기준). v2에서 50MB로 복귀 옵션을 남겨둔다.
4. **서버 의존 최소화** — 모든 핵심 기능은 오프라인에서 동작. 클라우드는 옵트인 동기화에만 사용.
5. **로컬 설치형** — 설치하면 외부 서버 없이 바로 돌아간다.
6. **멀티 프로바이더** — 단일 벤더에 종속되지 않고 여러 AI 서비스를 통합 수집. 새 프로바이더 추가는 어댑터 1개 작성으로 끝나는 구조.

### 7.1 코어 루프

사용자가 AI 서비스를 호출하면 다음 일이 일어난다.

1. 수집기(프록시 또는 로그 감시기)가 토큰 사용량을 캡처한다.
2. 원시 토큰을 저장하고, 게임용 단위인 **TokiPoint(TP)**로 환산한다.
3. 펫의 누적 TokiPoint가 늘고, 펫이 활기를 띤다.
4. 임계치에 도달하면 진화 이벤트가 발생한다.
5. 누적 TokiPoint가 충분히 쌓이면 새 펫을 뽑을 수 있다.

### 7.2 펫 시스템

#### 진화

종(species)마다 3단계 진화 트리를 가진다: 알(egg) → 어린이(child) → 성체(adult). 일부 희귀 종은 4단계까지 간다.

진화 조건은 단순한 누적 경험치(EXP)에 종별 추가 조건이 붙는다. 예를 들어 "특정 모델을 N회 호출", "캐시 히트율 X% 이상" 같은 조건이다. 진화 순간에는 풀스크린이 아니라 펫 윈도우 안에서 짧은 전환 애니메이션이 재생된다.

#### 상태(Mood)

펫은 사용 패턴에 따라 다섯 가지 상태를 오간다.

- **Idle** — 토큰이 흐르지 않을 때. 기본 호흡 애니메이션.
- **Excited** — 최근 1분 안에 대량 토큰이 들어왔을 때. 활기찬 모션과 파티클.
- **Sleepy** — 24시간 이상 사용이 없을 때. 졸린 표정과 슬로 모션.
- **Hungry** — 7일 이상 사용이 없을 때. 시들한 표정. 단, 펫이 죽지는 않는다(회복형).
- **Curious** — 사용자가 클릭하거나 드래그할 때. 시선이 따라가고 반응 모션이 나온다.

#### 인터랙션

- **클릭**: 펫이 시선을 향하고 짧은 모션(점프, 윙크 등)을 한다.
- **드래그**: 펫의 위치를 옮길 수 있다. 끌려가는 모션이 나온다.
- **우클릭 또는 트레이**: 컨텍스트 메뉴(인벤토리, 뽑기, 설정, 숨기기, 종료).
- **더블클릭**: HUD를 토글한다. 오늘 사용 토큰, 누적, 다음 진화까지 진행도가 표시된다.

### 7.3 뽑기 (Gacha)

가차의 재화는 **누적 TokiPoint**다. v1에서는 모든 프로바이더의 모든 토큰(입력·출력·캐시)이 **1:1로 동등하게 합산**된다. 단순하고 투명하게.

- **1회 비용** (초안, 밸런싱 대상): 100K TokiPoint = 1뽑
- **레어도**: Common · Rare · Epic · Legendary (확률 80 · 15 · 4 · 1, v1 초안)
- **천장(Pity)**: 50뽑을 모아도 Epic 이상이 안 나오면 자동 확정
- **중복 처리**: 같은 종이 또 나오면 그 종의 진화 가속 토큰을 받는다
- **활성 펫**: 보유 중인 펫 중 한 마리만 데스크탑에 띄운다. 인벤토리에서 교체.

**추가로 검토 중**: 한 주에 3개 이상 프로바이더에서 토큰이 발생하면 가차 확률에 살짝 buff가 붙는 "프로바이더 다양성 보너스". 단일 벤더 락인을 피하자는 제품 가치와 게임 메타를 정렬하는 장치.

### 7.4 HUD (필요할 때만 노출)

기본은 숨겨져 있고 마우스 hover 시 fade-in되는 미니 정보 패널.

- 오늘 사용 토큰 / 누적 토큰
- 다음 진화까지 게이지
- 뽑기 가능 횟수

### 7.5 시스템 트레이 (메뉴바)

펫 종에 따라 변하는 TokiMon 아이콘과 함께, 다음 메뉴를 제공한다.

Show/Hide Pet · Inventory · Gacha · Stats · Settings · Quit

트레이 아이콘을 클릭하면 누적 토큰과 오늘 사용량의 미니 요약이 나온다.

### 7.6 설정

- 자동 시작 on/off
- Always-on-top on/off (작업이 방해될 때 끌 수 있도록)
- 펫 윈도우 크기와 투명도
- 프록시 포트 변경
- 클라우드 동기화 로그인/해제
- 데이터 export(CSV·JSON), 데이터 초기화

### 7.7 클라우드 동기화 (옵트인)

여러 기기에서 펫을 같이 키우고 싶은 사용자용 기능. 명시적으로 켜야만 동작한다.

- **계정**: 이메일 매직링크 (Supabase Auth 등)
- **동기화 대상**: 보유 펫, 누적 토큰, 가차 천장 카운터
- **충돌 해소**: last-write-wins, 단 토큰 누적은 max() 머지
- **동기화하지 않는 것**: 토큰 이벤트 로그 자체 (프라이버시와 용량 둘 다)

### 7.8 추가로 검토 중인 기능

| 기능 | 설명 | 우선순위 |
|---|---|---|
| 프로바이더/모델별 펫 친화도 | Claude Opus 헤비유저, OpenAI 헤비유저, 로컬 LLM 헤비유저마다 다른 종이 빠르게 진화. "멀티벤더 유저 한정 펫" 같은 수집욕 자극 | P2 |
| 데일리 챌린지 | "오늘 캐시 히트 50% 이상" 같은 미션 → 가차 토큰 보너스 | P2 |

## 8. 성공 지표

v1 출시 후 한 달 시점에 다음을 본다.

- **W1 retention ≥ 40%** · **W4 retention ≥ 20%**
- **DAU/MAU ≥ 0.4** (매일 쓰는 사용자가 전체의 40% 이상)
- 사용자당 평균 보유 펫 수 **≥ 3**
- 프록시 통과 안정성: 사용자가 직접 API를 호출하는 것 대비 **latency 추가 < 20ms (p95)**
